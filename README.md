# 💰 Finance App

Удобный трекер расходов и доходов с фичей накоплений: ставишь финансовые цели, видишь график пополнений и получаешь напоминания, если за день не отложил ни рубля.

Тёмная тема, дизайн на основе системы Binance (единственный акцент — жёлтый, торговые зелёный/красный для направлений), адаптив под мобилку.

---

## ✨ Возможности

### Трекер расходов
- Учёт расходов и доходов с категориями, заметками и датой
- Сводка: баланс, доходы, расходы, остаток месячного бюджета
- График доходов/расходов по месяцам (area chart)
- Разбивка расходов по категориям (donut chart) с процентами
- Лента операций с группировкой по дням и фильтрами (тип + категория)
- Месячный бюджет с предупреждением при перерасходе

### Накопления (цели)
- Создание целей: сумма, иконка, цвет, дедлайн, время напоминания
- Прогресс-бар и процент по каждой цели
- Быстрое пополнение (+500 / +1000 / +5000 / +10000 или своя сумма)
- **Подсказка «сколько откладывать в день»**, чтобы успеть к дедлайну
- График динамики накоплений по всем целям (cumulative line chart)
- Бейдж «Готово» при достижении цели

### Напоминания
- Если за день цель не пополнена — после заданного времени прилетает push-уведомление (Web Notifications API)
- Баннер на дашборде со списком целей, ждущих пополнения сегодня
- Тумблер включения/выключения напоминаний в настройках

### Прочее
- Настройки: имя, валюта (₽/$/€/₸), месячный бюджет
- Данные сохраняются между сессиями (localStorage через mock-API)
- Полностью адаптивная вёрстка: сайдбар на десктопе → нижняя навигация на мобилке

## 🔬 Research: что есть в подобных приложениях

Прежде чем добавлять функционал, я посмотрел, из чего состоят популярные трекеры (CoinKeeper, YNAB, Money Lover, Monefy, Spendee, Wallet by BudgetBakers). Сводка решений и что из этого вошло в проект:

| Паттерн | Где встречается | Статус в проекте |
|---|---|---|
| Быстрый ввод операции с категориями-иконками | Monefy, CoinKeeper | ✅ реализовано |
| Donut-разбивка расходов по категориям | Monefy, Spendee | ✅ реализовано |
| Месячный бюджет + предупреждение о перерасходе | YNAB, Wallet | ✅ реализовано |
| Цели накоплений с прогрессом | Money Lover, Wallet | ✅ реализовано |
| «Сколько откладывать, чтобы успеть к сроку» | редко, премиум-фича | ✅ реализовано |
| Напоминания/стрик пополнений | привычковые трекеры | ✅ push-уведомления |
| Тренд доходов/расходов по месяцам | все | ✅ реализовано |
| Мультивалютность | Spendee, Wallet | ⚙️ переключатель валюты (без конвертации курсов) |
| Регулярные/повторяющиеся платежи | YNAB, Wallet | 🔜 в бэклоге |
| Импорт из банка / OFX-CSV | YNAB, Spendee | 🔜 в бэклоге |
| Совместный бюджет (семья) | Spendee, Wallet | 🔜 в бэклоге |

### Идеи для развития (бэклог)
- Повторяющиеся транзакции (подписки, зарплата) с автодобавлением
- Стрик «дней подряд с пополнением» и геймификация целей
- Service Worker + Push API, чтобы напоминания работали при закрытой вкладке
- Импорт CSV/банковской выписки и категоризация
- Аналитика: средний чек, топ категорий месяца, прогноз к концу месяца

## 🧱 Архитектура — Feature-Sliced Design (FSD)

Проект организован по [FSD](https://feature-sliced.design): слои выстроены по уровню ответственности, импорты идут только «сверху вниз» (`app → pages → widgets → features → entities → shared`).

```
src/
├── app/          # инициализация: провайдеры, роутер, layout, глобальные стили
├── pages/        # страницы-композиции (dashboard, transactions, savings, settings)
├── widgets/      # самостоятельные блоки UI (sidebar, header, графики дашборда)
├── features/     # пользовательские сценарии (добавить операцию, создать цель,
│                 #   пополнить, напоминания, фильтры) — формы RHF + Zod
├── entities/     # бизнес-сущности: модель + запросы + селекторы + их UI
│   ├── transaction/   # useTransactions, селекторы аналитики, TransactionItem
│   ├── goal/          # useGoals, useContribute, логика напоминаний, GoalCard
│   └── account/       # Zustand-стор настроек
└── shared/       # переиспользуемое: UI-кит, lib-утилиты, mock-API, конфиг, типы
```

### Почему так
- **Разделение клиентского и серверного состояния.** Настройки пользователя живут в Zustand (`entities/account`), а данные операций/целей — в TanStack Query (`entities/*/queries.ts`). Это убирает típичную путаницу «всё в одном глобальном сторе».
- **Mock-API вместо моков прямо в компонентах.** `shared/api/db.ts` имитирует бэкенд с задержкой и сохраняет данные в localStorage. Когда появится реальный сервер — меняется только этот файл, хуки и компоненты не трогаются.
- **Селекторы отдельно от запросов.** Вся аналитика (тоталы, разбивка по категориям, тренды, прогресс целей, «сколько в день») — чистые функции в `selectors.ts`, их легко тестировать.
- **Формы изолированы в features** со своими Zod-схемами — переиспользуются и на дашборде, и на страницах.

## 🚀 Запуск

```bash
npm install
npm run dev        # http://localhost:5173
```

Другие команды:

```bash
npm run build      # production-сборка
npm run preview    # предпросмотр сборки
npm run typecheck  # проверка типов
npm run lint       # ESLint
```

> При первом запуске приложение наполняется демо-данными (несколько операций и целей), чтобы графики не были пустыми. Очистить — удалить ключи `fa.*` в localStorage.

## 🎨 Дизайн-система

Палитра и токены вынесены в `tailwind.config.ts` на основе анализа дизайн-системы Binance:

- **Акцент:** Binance Yellow `#fcd535` — только для основных действий и брендовых акцентов
- **Холст:** near-black `#0b0e11`, карточки `#1e2329`, hairline `#2b3139`
- **Семантика направления:** зелёный `#0ecb81` (доход/рост), красный `#f6465d` (расход/падение) — только для цифр и направления, не как фон карточек
- **Шрифты:** Inter (текст) + IBM Plex Mono (цифры, табличные суммы) как open-source замена BinanceNova/BinancePlex
